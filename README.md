This repository/tool enables, for example, the signing/sealing of payments and consents according to Berlin Group standards. The focus is on JSON.
XML (ISO 20022) is also supported proactively through the use of the DSS (however, this must currently be marked as proprietary, as it is not yet described in detail in the Berlin Group specification).


Detailed documentation of the tool can be found in the "docs" folder.

The "examples" folder contains application examples as well as scripts for creating your own certificates and key materials.

The "myopenfinanceservices.com.SigningAuthorisation.examples" package contains two classes that you can use as a starting point for your implementation.


Information about the Berlin Group brand and its standardization and harmonization activities can be found at: https://www.berlin-group.org
